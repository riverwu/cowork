# SlideML Theme Design Guide

How to design a new SlideML theme. Distilled from the Anthropic Pptx
skill's design notes plus what we learned shipping `technical-blue`,
`editorial-warm`, `midnight-executive`, `forest-moss`, and
`charcoal-minimal`.

> "Plain bullets on a white background won't impress anyone."
> — Pptx skill, *Design Ideas*

---

## 1. Contrast — the load-bearing rule

Every load-bearing token pairing must clear WCAG AA. The theme loader
audits this on `loadTheme()` and warns to stderr; opt into hard
enforcement with `theme.style.contrastTarget = "AA"` (or `"AAA"`).

| Pairing | Threshold (AA) | Why |
|---|---|---|
| `text-strong` on `bg-canvas` | ≥ 4.5 | Body text on slide |
| `text-strong` on `bg-card` | ≥ 4.5 | Body text on cards |
| `text-muted` on `bg-canvas` | ≥ 4.5 | Subtitles, captions |
| `brand-primary` on `bg-canvas` | ≥ 3.0 | KPI value, accent text (large) |
| `accent` on `bg-canvas` | ≥ 3.0 | Decorative shapes |
| best-of(`text-strong`, white) on `brand-deep` | ≥ 4.5 | Closing/section panels |

Tools:
- `slideml audit <deck.pptx>` doesn't currently check rendered contrast,
  only OOXML conformance. The audit lives at theme-load time.
- The `bestTextOn(ctx, bgColor)` primitive auto-picks white vs theme
  text-strong for full-bleed brand-color panels — use it whenever a
  layout paints text directly on a brand color.

## 2. Color philosophy — content-informed, dominant

Don't default to generic blue. Pick a palette that *means* something for
the deck's domain:
- **Sustainability** → forest + moss + cream (`forest-moss`)
- **Executive board** → midnight + ice + amber (`midnight-executive`)
- **Engineering / data** → navy + cyan (`technical-blue`)
- **Editorial / consulting** → cream + rust + gold (`editorial-warm`)
- **Print, minimal** → off-white + near-black + warm pop (`charcoal-minimal`)

**Dominance over equality.** One color carries 60-70% of the visual
weight (canvas), one is the primary accent (brand-primary), one is a
sharp pop (accent). Never give all colors equal weight.

**Limit the palette.** ≤ 5 hues per slide. The token system enforces
this: there are only 8 color tokens total.

## 3. Typography — pair with intent

| Header font | Body font | Use when |
|---|---|---|
| `Inter` | `Inter` | Engineering / dashboard (technical-blue, midnight-executive) |
| `Source Serif Pro` | `Source Sans Pro` | Editorial, consulting (editorial-warm) |
| `Source Sans Pro` | `Source Sans Pro` | Minimal, print (charcoal-minimal) |
| `Georgia` | `Calibri` | Classical / academic |
| `Trebuchet MS` | `Calibri` | Friendly / consumer |

Set both faces in `theme.json → oxml.fontScheme`. Inter (when used
twice) is fine — but only because Inter's variable weights give a real
hierarchy via weight, not just family.

**Size hierarchy** (already baked into primitives, override via opts):
- Slide title: 44pt bold (`slideTitle` default)
- Section header: 24-32pt bold
- Body: 14-16pt
- Captions: 10-12pt muted

The `2:1 title-to-body` ratio is non-negotiable.

## 4. AI-tells to avoid

The Pptx skill calls out specific patterns that scream "this was
generated by an LLM":

1. **Accent rules under titles.** Universal cyan/brand stripes under
   every title look templated. → Set `style.titleAccentRule: false` on
   restrained themes (`editorial-warm`, `forest-moss`, `charcoal-minimal`).
   Keep `true` only for high-density / engineering themes where the
   stripe reads as visual rhythm (technical-blue, midnight-executive).

2. **Centered body text.** Always left-align body and lists. Center only
   titles, sometimes subtitles. Never center bullet lists.

3. **Same layout repeated 8 slides in a row.** Vary: cover → KPI grid →
   chart → bullet+image → section → table → compare → closing. This is
   why we ship 17 layouts.

4. **Equal-weight palettes.** See §2 above.

5. **Generic blue everywhere.** See §2 above.

6. **Text-only slides.** Every content slide should have at least one
   visual element (KPI value, chart, image, accent shape).

## 5. Visual motif — pick one and commit

Stage B's primitives give themes a way to commit to a motif without
rewriting layouts:
- **`titleAccentRule: true`** — accent stripes are your motif
  (technical-blue, midnight-executive)
- **`titleAccentRule: false`** — clean type sets the rhythm; rely on
  weight + size hierarchy alone (editorial-warm, charcoal-minimal)
- Card stripes — `card({ accentStripe: "brand-primary" })` for KPI tiles
- Brand-deep panels — closing / section-divider for visual closure

Don't mix motifs. If you set `titleAccentRule: false`, layouts that
need extra emphasis should reach for size or weight, not introduce a
sneaky stripe somewhere else.

## 6. Theme manifest checklist

A complete `theme.json` declares:

```jsonc
{
  "name":           "...",       // kebab-case, matches dir name
  "displayName":    "...",        // human-friendly, used in PowerPoint
  "tokens": {                    // 8 colors + 3 font stacks
    "bg-canvas": "...", "bg-card": "...",
    "text-strong": "...", "text-muted": "...",
    "brand-primary": "...", "brand-deep": "...",
    "accent": "...", "divider": "...",
    "font-latin": [...], "font-cjk": [...], "font-mono": [...]
  },
  "style": {
    "titleAccentRule": true|false,
    "contrastTarget": "warn"|"AA"|"AAA"
  },
  "oxml": {                      // PowerPoint color/font picker
    "clrScheme": { ... 12 slot map ... },
    "fontScheme": { "majorLatin": "...", "minorLatin": "..." }
  },
  "layouts": [...],              // declared in theme.md too
  "chrome":  ["page-number", ...]
}
```

`theme.md` mirrors the structure: every layout in `theme.json` must have
a matching `### name` subsection with a `- slot` line and a thumbnail.
Add `**Guidance:**` blocks to layouts where content quality matters
(stat-grid-3, chart-with-takeaway, dashboard) — agents read these.

## 7. Adding a new theme — checklist

1. `mkdir slideml/src/themes/<name>/{layouts,chrome,thumbnails}`
2. Copy 6 essential layouts from technical-blue (cover, agenda,
   stat-grid-3, chart-with-takeaway, bullet-with-image, closing) plus
   `chrome/page-number.ts`. Layouts are theme-portable — they read
   tokens from `ctx.color()`, so just changing the token table changes
   the look.
3. Generate placeholder thumbnails (any 1×1 PNG works; replace later).
4. Write `theme.json` (palette + style + oxml + layouts).
5. Write `theme.md` (when-to-use, layout reference with slot lists).
6. Build (`pnpm run build`) and run any `slideml layouts --theme <name>`
   command. Stderr will show contrast warnings — fix them.
7. Compile a 5-slide showcase (cover → KPI → chart → bullets → closing)
   and visually QA in PowerPoint.
8. If passing all WCAG AA, ship. If not, darken the offender.

---

## 8. What this guide deliberately doesn't cover

- **Animation / transitions.** Out of scope; agents rarely need them.
- **Custom shape geometries.** Use the existing primitives; richer
  shapes belong in components, not in theme tokens.
- **Per-slide overrides of palette.** Use `deck.background.color` for
  one-off slides; don't multiply themes.
- **Brand logos.** Embed via `image-ref` slots; out of scope for the
  theme system (logos belong to a deck, not a theme).
